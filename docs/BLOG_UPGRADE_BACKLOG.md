| 搁置 | 高 | 歌曲管理从 TOML 升级为更结构化的数据源 | 当前 `data/music.toml` 已可用，但歌曲数量增加后，可以考虑拆分为每首歌一个文件，降低合并冲突和编辑风险。 |

| 保留 | 中 | 歌词面板视觉状态细化 | 可继续优化长歌词、制作信息、英文歌词、纯音乐占位的显示规则。 |
| 待审计 | 高 | 当前播放唯一主舞台 | 音乐页可以从“每张卡各自展开”升级为一个主舞台，点击不同歌曲时舞台内容切换，整体更像专属音乐房间。 |

| 待审计 | 中 | 记住歌词打开状态 | 目前歌词开关是手动按钮控制，后续可用 `localStorage` 记住用户上次是否打开歌词。 |

| 保留 | 低 | Apple Music 元数据辅助脚本 | 给定歌名和歌手后自动查询 artwork、preview、link，减少手工录入。 |
